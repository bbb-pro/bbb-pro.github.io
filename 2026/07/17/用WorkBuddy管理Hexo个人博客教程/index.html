用WorkBuddy管理Hexo个人博客教程

发表于2026-07-17|更新于2026-07-17|教程
|浏览量:

适合人群：已经用 Hexo + Butterfly 主题搭建了博客、部署在 GitHub Pages 上的开发者
工具版本：WorkBuddy（CodeBuddy Desktop）+ Hexo 6.x + Butterfly 4.x
预计阅读：8 分钟

为什么用 WorkBuddy 管理博客？

写博客最痛苦的不是写文章，而是——

  • 打开终端 → cd 到博客目录 → hexo new → 打开编辑器 → 写完 → hexo clean && hexo g && hexo d → 推送源码……这一套流程每次都要走一遍
  • 想改个主题配置，翻了半天文档找不到参数
  • 想定时发布，得自己写 CI 脚本
  • 换台电脑，环境都没配，写不了博客

WorkBuddy 解决的就是这些问题：它能直接在你的博客项目目录里读写文件、执行命令、运行自动化任务。你只需要用自然语言说”帮我写一篇关于 XXX 的文章并发布”，它就能帮你完成从创建到部署的全部操作。

下面分三个场景讲清楚：对话式写作 + 一键部署、已有文章的发布与管理、自动化运维。

前置准备

1. 确保 WorkBuddy 能访问你的博客目录

在 WorkBuddy 中打开你的 Hexo 博客项目目录（就是包含 _config.yml、source/、themes/ 的那个文件夹）。WorkBuddy 会以这个目录为工作区根目录。

2. 确认本地 Hexo 环境

跟 WorkBuddy 说：

帮我检查一下当前目录的 Hexo 环境是否正常，看看 package.json 里的依赖和 _config.yml 的部署配置

WorkBuddy 会读取你的 package.json、_config.yml、_config.butterfly.yml，确认环境状态，并告诉你是否有问题。

这是最常用的方式——在 WorkBuddy 对话框里直接写文章，写完让它帮你部署。

Step 1：创建新文章

直接跟 WorkBuddy 说：

帮我新建一篇博客，标题是”用 Docker Compose 搭建本地开发环境”，分类是”DevOps”，标签加上 docker、开发环境

Step 2：在对话中直接写正文

你可以用两种方式让 WorkBuddy 帮你写正文：

方式 A：给大纲，让它扩写

帮我按以下大纲写正文：

  2. 安装和配置
  3. 编写 docker-compose.yml

WorkBuddy 会直接把内容写入文章文件，你只需审阅和修改。

场景二：已有文章的发布与管理

如果你已经有一批 Markdown 文章（比如从其他平台迁移、或者之前用别的编辑器写的），WorkBuddy 可以帮你批量导入、优化和管理。

1. 导入已有文章

把你的 Markdown 文件放到 source/_posts/ 目录下，然后跟 WorkBuddy 说：

帮我扫描 source/_posts/ 目录下的所有文章，检查每篇的 Front Matter 是否完整，缺什么补什么

WorkBuddy 会逐个检查每篇文章的 Front Matter，自动补全缺失的 title、date、categories、tags 等字段。

2. 批量优化文章格式

场景三：自动化运维

这是 WorkBuddy 最强大的能力——用自动化任务让博客自己跑起来。

1. 定时自动备份源码

防止文章丢失，每天自动把博客源码推送到 Git 仓库：

帮我创建一个自动化任务，每天凌晨 2 点自动把博客源码 commit 并 push 到 GitHub

WorkBuddy 会创建一个 recurring 自动化任务，从此你的博客源码每天都会自动备份。

2. 定时生成草稿选题

让 WorkBuddy 每周帮你生成博客选题灵感：

帮我创建一个自动化任务，每周一早上 9 点，根据我博客已有的文章分类和最近的技术热点，推荐 3 个博客选题

3. 定时检查博客健康状态

帮我创建一个自动化任务，每周日检查一下博客的 GitHub Pages 是否正常访问、有没有死链

WorkBuddy 会在每周日访问你的博客首页确认可访问性，扫描文章中的外链并生成健康报告。

4. 自动发布队列

如果你习惯周末集中写文章、工作日定时发布：

帮我创建一个自动化任务，每周三和周五各发布一篇 source/_drafts/ 目录下最早的那篇草稿

这样你只需周末把文章写好放到草稿箱，工作日自动发布，保持更新频率。

常见问题

Q：WorkBuddy 执行 hexo 命令时报错怎么办？

跟 WorkBuddy 说：”执行 hexo 命令报错了，帮我看看错误信息”。它会读取报错内容，分析原因（通常是依赖没装、Node 版本不对、配置文件语法错误等），并给出修复方案。

Q：换了一台电脑，博客环境还没配，能用 WorkBuddy 吗？

可以。在新电脑上安装 WorkBuddy 后，打开一个空目录，跟它说：

帮我从 GitHub 克隆博客仓库，并安装 Hexo 环境依赖

WorkBuddy 会执行 git clone、npm install，帮你快速恢复写作环境。

Q：WorkBuddy 写的文章质量怎么样？需要自己改吗？

WorkBuddy 生成的是高质量初稿，建议你：

  • 技术文章：核心代码和命令一定要自己验证一遍
  • 观点文章：加入你自己的经验和案例，让内容更有个人风格
  • 翻译文章：检查专业术语是否准确

把 WorkBuddy 当成你的”博客助手”而不是”代笔”——它帮你搞定 80% 的框架和初稿，你负责 20% 的灵魂注入。

Q：自动化任务执行失败了会通知我吗？

会的。WorkBuddy 的自动化任务执行后，你会在对话中收到执行结果通知。如果任务失败，它会告诉你失败原因。建议把重要的自动化任务设置为推送到微信/企业微信，这样即使没打开 WorkBuddy 也能收到通知。

总结

用 WorkBuddy 管理 Hexo 博客，本质上就是把原来需要记的命令、需要开的终端、需要手动跑的流程，全部交给 AI 自动化。你只需关注两件事：

  1. 写什么（选题和内容）
  2. 审一下（发布前过目）

剩下的——创建文件、格式化 Front Matter、构建部署、源码备份、定时发布、健康检查——全部交给 WorkBuddy。

现在就开始：打开 WorkBuddy，指向你的博客目录，说一句”帮我检查一下博客环境”，你的自动化博客管理之旅就开始了。
